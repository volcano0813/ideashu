# 小红书内容风格指南

## 平台内容规范

### 字数
- 推荐范围：300-800 字
- 标题：不超过 20 字（含 emoji）
- 每段不超过 3 行（手机端显示）

### Emoji 使用
- 密度：每 100 字 1-2 个
- 不要每段开头都放 emoji（AI 味明显）
- 随机分布，有的段落不放
- 避免堆砌：同一句不超过 2 个 emoji

### 段落
- 短段落为主，2-3 句话一段
- 段与段之间空一行（小红书会自动加间距）
- 长段落不超过 5 行

### 标签
- 数量：5-10 个
- 构成：2-3 个大流量词 + 3-5 个精准长尾词 + 1-2 个品牌/IP 词
- 放在正文最后，用 # 号

## 内容结构模板

### 模板 A：种草推荐
```
[emoji] [标题：数字 + 关键词 + 情绪词]

[开头钩子：1-2 句，制造好奇或共鸣]

[个人背景/使用场景：为什么我会接触到这个]

[核心推荐内容：2-3 个要点，每个要点附带个人感受]

[真实使用细节：一个具体的场景或数据]

[收尾：自然的总结或互动引导]

#标签1 #标签2 ...
```

### 模板 B：经验分享
```
[标题：问题/痛点 + 解决方案暗示]

[痛点共鸣：你是不是也遇到过……]

[我的经历：具体的故事，有时间、地点、细节]

[解决方案：我怎么做的，效果如何]

[注意事项：1-2 个真实踩过的坑]

[互动：你们有类似的经历吗？]

#标签1 #标签2 ...
```

### 模板 C：探店/测评
```
[标题：地点/产品 + 结论前置]

[时间地点：什么时候去的/什么时候买的]

[第一印象：环境/外观/开箱]

[核心体验：重点描述 2-3 个维度]

[意外发现：一个没预料到的点（好的或不好的）]

[结论：推不推荐，适合什么人]

#标签1 #标签2 ...
```

## 标题公式

- 数字 + 关键词："5 个让你早起不痛苦的方法"
- 人群 + 痛点："上班族必看的快手早餐"
- 反差/对比："月薪 5000 vs 50000 的通勤包"
- 问句："为什么你的手冲咖啡总是发酸？"
- 场景 + 结果："下班后做了这件事，整个人都不一样了"
